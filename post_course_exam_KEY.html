Post-course exam — answer key

Worked solutions and scoring rubrics for the FW 536 Day-5-afternoon post-course exam. Pair with pre_course_exam_KEY.html when computing class-wide gain.

Scoring guidance

Unlike the pre-course exam, the post-course exam is graded on its merits — students have had five days of intensive instruction and should be expected to demonstrate understanding. Use the standard partial-credit rubric:

  • Full credit: correct answer with correct reasoning.
  • Partial credit: correct setup with calculation/interpretation error.
  • Generous partial credit for the applied case study (G1) — there is no single right answer; reward clarity of reasoning.

Total: 50 points. Class median goal: 35–40 / 50 (= 70–80%). If median is < 30 or > 45, recalibrate questions for the next offering.

Part A — Probability fundamentals (6 points)

A1. Bayes' rule applied (2 pts)

P(present | +) = (0.90 × 0.10) / (0.90 × 0.10 + 0.04 × 0.90) = 0.09 / (0.09 + 0.036) = 0.714.

(0.90 * 0.10) / (0.90 * 0.10 + 0.04 * 0.90) # 0.7143

A2. Updating from two samples (2 pts)

Update twice. After first positive, prior becomes 0.714. After second positive: (0.90 × 0.714) / (0.90 × 0.714 + 0.04 × 0.286) = 0.982.

p1 <- (0.90*0.10)/(0.90*0.10 + 0.04*0.90)
(0.90*p1) / (0.90*p1 + 0.04*(1 - p1)) # 0.9824

A3. Joint probability with imperfect detection (2 pts)

P(zero detections) = P(absent) + P(present) × (1 − 0.25)³ = 0.60 + 0.40 × 0.4219 = 0.60 + 0.169 = 0.769.

0.60 + 0.40 * (1 - 0.25)^3 # 0.7688

Common partial-credit error: just computing (1 − 0.25)³ = 0.422, ignoring the prior on presence. Give 1 pt.

Part B — Distributions (6 points)

B1. Pick the distribution and why (2 pts)

Negative Binomial. Variance > mean, so the data are overdispersed; the Poisson constrains variance = mean and would underestimate uncertainty. NB has a dispersion parameter that absorbs the extra variance.

B2. Beta-Binomial (2 pts)

Camera-to-camera heterogeneity puts some cameras at very low p (essentially zero detections all season) and others at higher p. Compared to a single shared p, the mixture concentrates probability on the extremes — including "all zero detections." This is exactly the mechanism behind the Beta-Binomial.

B3. Continuous distribution with a bound (2 pts)

Beta. Support is exactly (0, 1) — matches percent cover; fit with betareg in R.

Part C — Regression and GLMs (10 points)

C1. Log-link biology (3 pts)

a. exp(−0.18) = 0.835. Each 1 °C of warming multiplies expected counts by 0.835, i.e., a ~17% drop per degree.

b. Half-life: −ln(2) / −0.18 = ln(2) / 0.18 = 3.85 °C. So roughly 3.9 °C of warming halves expected counts.

C2. Logit-link biology (3 pts)

a. p = plogis(−3) = 0.0474.

b. p = 0.5 when log-odds = 0, so 0.05 · x = 3 → x = 60.

c. Marginal at canopy = 60: β · p · (1 − p) = 0.05 · 0.5 · 0.5 = 0.0125. So a 1-percentage-point canopy increase raises p by about 0.013 (about 1.3 percentage points) at canopy = 60.

C3. Overdispersion diagnosis (2 pts)

Ratio 412 / 80 = 5.15. Severely overdispersed. Switch to Negative Binomial (MASS::glm.nb) or to quasi-Poisson if only point estimates and inflated SEs are needed.

C4. Interaction interpretation (2 pts)

Temp effect on trawl = −0.10 + 0.15 = +0.05. exp(0.05) = 1.051. Trawl catches increase ~5% per °C warming, opposite the gillnet pattern (which decreases ~10% per °C).

Part D — Mixed models / hierarchy (8 points)

D1. Notation (2 pts)

$y_{ij} = \beta_0 + \beta_1 x_{ij} + a_i + \varepsilon_{ij}$, where $a_i \sim \mathcal{N}(0, \sigma^2_{\text{site}})$ and $\varepsilon_{ij} \sim \mathcal{N}(0, \sigma^2)$, independently. Or equivalently $y_{ij} \sim \mathcal{N}(\beta_0 + \beta_1 x_{ij} + a_i, \sigma^2)$ with $a_i \sim \mathcal{N}(0, \sigma^2_{\text{site}})$.

Scoring: 2 pts for any correct version. 1 pt for the random-intercept idea but missing distributional assumptions.

D2. REML vs ML (3 pts)

ML estimates variance components downward-biased because it doesn't account for the degrees of freedom consumed by estimating fixed effects. REML maximizes the likelihood of the residual variance after fixed-effect estimation, removing the bias. Rule: REML estimates compare across random-effect structures (same fixed structure); ML estimates compare across fixed-effect structures (because REML likelihoods aren't comparable when fixed effects differ); report REML estimates.

D3. Random slopes (3 pts)

Var(intercept) = 4.2 (sd ≈ 2.05) — substantial between-site variation in baseline.

Var(slope) = 0.05 (sd ≈ 0.22) — moderate between-site variation in temperature response.

Correlation = −0.6 means that sites with higher baselines (intercepts) tend to have shallower (more negative or less positive) slopes. Biologically: sites that start with more of the response tend to respond less strongly to temperature — a ceiling effect, density-dependence, or habitat saturation could produce this pattern.

Part E — Maximum likelihood (6 points)

E1. MLE by hand (3 pts)

Log-likelihood: log L = k log p + (n − k) log(1 − p) = 12 log p + 28 log(1 − p).

d log L / dp = 12 / p − 28 / (1 − p). Set to 0: 12 (1 − p) = 28 p → 12 = 40 p → p̂ = 12/40 = 0.30.

Scoring: 3 pts for full derivation. 2 pts if they jump to 12/40 with brief justification. 1 pt for correct answer with no work.

E2. LRT and profile-likelihood CI (3 pts)

LRT: $\Lambda = 2 (\log L_{\text{full}} - \log L_{\text{reduced}})$ has an asymptotic $\chi^2_{df}$ distribution under the null, where df = difference in number of parameters.

Profile CI: for parameter θ, fix θ to candidate values, maximize log L over the other parameters, and find the range of θ where $\log L(\theta) \ge \log L_{\hat\theta} - 1.92$. The 1.92 comes from $\frac{1}{2} \chi^2_{0.95, 1} = \frac{1}{2}(3.841) = 1.92$, the LRT cutoff for a 95% CI on a single parameter.

Part F — Bayes (8 points)

F1. Conjugacy (3 pts)

Posterior: Beta(2 + 18, 8 + 12) = Beta(20, 20). Mean = 20 / (20 + 20) = 0.5. (Compare to MLE of 18/30 = 0.6 — the prior pulls toward 2/(2+8) = 0.2.)

F2. BCI vs HPDI (2 pts)

BCI: equal-tail (e.g., 2.5%–97.5%) quantile interval. HPDI: the shortest interval containing 95% of posterior mass. They coincide for symmetric posteriors but diverge for skewed ones, where HPDI is more meaningful.

F3. Sketch a Nimble model (3 pts)

nimbleCode({
 for (i in 1:n) {
 log(lambda[i]) <- beta0 + beta1 * x[i]
 y[i] ~ dpois(lambda[i])
 }
 beta0 ~ dnorm(0, sd = 10)
 beta1 ~ dnorm(0, sd = 10)
})

Scoring: 3 pts for clean code resembling above. 2 pts for correct structure but missing detail (e.g., no priors). 1 pt for pseudo-code that shows understanding of likelihood + priors. Accept either sd= or precision parameterization (tau) — both are valid Nimble.

Part G — Applied case study (6 points)

G1. From question to model

A model answer covers:

  • Distribution and link: Negative Binomial with log link (variance > mean by a factor of ~4). Poisson would underestimate SEs. Reward students who note the overdispersion check.
  • Grouping: random intercept by stream (because surveys within streams are not independent); year may be modeled as a fixed effect (if 3 levels) or random (if treated as a sample from years). Survey is the observational unit.
  • Fit check: residual deviance / df near 1; DHARMa residual simulation; posterior predictive check if Bayesian; plot of residuals vs fitted and vs predictors.
  • Frequentist vs Bayesian: reasonable arguments both ways. Frequentist (glmer.nb or glmmTMB) is faster; Bayesian (Nimble) is preferable if the manager wants probability statements about the management effect or if there are convergence/small-sample concerns with the variance components. Reward clarity of reasoning, not the specific choice.

Suggested rubric: 1.5 pts each for (a), (b), (c), (d). Generous partial credit.

Pre/post analysis to compute

  1. For each student: pre_pct, post_pct, gain = post_pct − pre_pct.
  2. Class-wide: mean gain, SD, paired t-test (or paired Wilcoxon).
  3. Per-topic gain analysis: which parts (A–G) showed the largest and smallest gains? Use this to recalibrate workshop pacing in subsequent offerings.
  4. Visual: a paired-difference plot (pre on x-axis, post on y-axis, points colored by gain). Add a y = x reference line — points above the line are students who learned.
