Exercise 1 — "Vague" priors on a log link

The point: a $\mathcal{N}(0, \sigma=100)$ prior on a regression coefficient is wildly informative once you exponentiate it.

set.seed(1)
beta <- rnorm(1e5, 0, 100)
eff_raw <- exp(beta * 3) # 3-degree latitudinal change
mean(abs(log10(eff_raw)) > 6) # ~ 0.98
eff_sc <- exp(beta * 1) # 1 SD on scaled x
mean(abs(log10(eff_sc)) > 6) # ~ 0.94

Interpretation. ~98% of the prior on the raw-scale model puts the implied effect outside a million-fold range — far beyond anything physically possible. Scaling reduces this slightly but not enough; you also need a tighter prior on $\beta$ itself. The fix that works in practice is both: scale predictors AND use $\mathcal{N}(0, \sigma \approx 2)$ priors.

Common mistake: Students who use a tighter prior but keep raw predictors often still see chains stall, because $\beta \cdot x$ can still get large when $|x|$ is large. The point is the product, not either piece alone.

Exercise 2 — Fit the ant model two ways

library(nimble); library(MCMCvis); library(coda)
ants <- read.csv("ants.csv")

antCode <- nimbleCode({
 for (i in 1:n) {
 ants[i,1] ~ dpois(lam[i])
 log(lam[i]) <- b0 + b1*ants[i,2] + b2*ants[i,3] + b3*ants[i,4]
 }
 b0 ~ dnorm(0, tau); b1 ~ dnorm(0, tau)
 b2 ~ dnorm(0, tau); b3 ~ dnorm(0, tau)
})

## (a) Raw predictors with very vague prior -- chains crawl
constsA <- list(ants=as.matrix(ants), n=nrow(ants), tau=1/10000)
out_a <- nimbleMCMC(antCode, constants=constsA,
 monitors=c("b0","b1","b2","b3"),
 nchains=3, niter=10000, nburnin=500,
 samplesAsCodaMCMC=TRUE)
gelman.diag(out_a) # often R-hat > 1.2

## (b) Scaled predictors, same prior -- chains converge instantly
ants$forest <- as.numeric(scale(ants$forest))
ants$latitude <- as.numeric(scale(ants$latitude))
ants$elevation <- as.numeric(scale(ants$elevation))

constsB <- list(ants=as.matrix(ants), n=nrow(ants), tau=1/10000)
out_b <- nimbleMCMC(antCode, constants=constsB,
 monitors=c("b0","b1","b2","b3"),
 nchains=3, niter=10000, nburnin=500,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(out_b)
glm(richness ~ ., data=ants, family=poisson)$coef # compare

## (c) Tighter prior
constsC <- list(ants=as.matrix(ants), n=nrow(ants), tau=1/4)
out_c <- nimbleMCMC(antCode, constants=constsC,
 monitors=c("b0","b1","b2","b3"),
 nchains=3, niter=10000, nburnin=500,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(out_c) # nearly identical to (b)

Expected posterior medians (scaled): $b_1$ (forest) ≈ 0.6 [0.4, 0.8], $b_2$ (latitude) ≈ -0.3 [-0.5, -0.1], $b_3$ (elevation) ≈ -0.4 [-0.6, -0.2]. These match glm() output to 2 decimals.

Instructor note. The "vague prior on raw scale" case often produces NaN log densities and a stalled chain. Show this live; it makes the point unforgettable.

Exercise 3 — Lizards: convergence, prediction, and prior sensitivity

library(nimble); library(MCMCvis); library(coda)
liz <- read.csv("IslandsLizards.csv")
x.scaled <- as.numeric(scale(liz$perimeterAreaRatio))

lizCode <- nimbleCode({
 a ~ dnorm(0, 1/2.71); b ~ dnorm(0, 1/2.71)
 for (i in 1:n) {
 logit(p[i]) <- a + b*x[i]
 y[i] ~ dbern(p[i])
 }
 x10 <- (10 - mx)/sx; x20 <- (20 - mx)/sx
 y10 <- ilogit(a + b*x10)
 y20 <- ilogit(a + b*x20)
 diff <- y20 - y10
})
consts <- list(n=nrow(liz), x=x.scaled,
 mx=mean(liz$perimeterAreaRatio),
 sx=sd(liz$perimeterAreaRatio))
samples <- nimbleMCMC(lizCode, constants=consts,
 data=list(y=as.integer(liz$presence)),
 monitors=c("a","b","y10","y20","diff"),
 nchains=3, niter=6000, nburnin=1000,
 samplesAsCodaMCMC=TRUE)
gelman.diag(samples[, c("a","b")])
heidel.diag(samples[, c("a","b")])
b.post <- as.matrix(samples)[, "b"]
mean(b.post < 0) # ~ 0.99+
MCMCsummary(samples, params=c("y10","y20","diff"))

Expected. R-hat ≈ 1.00 on all parameters. $P(b < 0) > 0.99$, confirming higher PA reduces occupancy. diff posterior median < 0 with 95% CI also negative.

Biology. Higher perimeter:area ratio means more edge habitat per unit interior, which on small or irregular islands corresponds to more predation pressure and less stable microhabitat — consistent with the lizards' lower occurrence probability there.

Instructor note. If a student reports R-hat > 1.05 for the random-slope model, they may need to lengthen niter to 50,000 and thin; this is a model where the default chain length sometimes isn't enough for the hyperparameters $\sigma_\alpha$ and $\sigma_\beta$ to converge.

Exercise 5 — Shrinkage by hand

Using $\hat\alpha_j = (n_j\bar y_j/\sigma^2 + \mu_\alpha/\sigma_\alpha^2)/(n_j/\sigma^2 + 1/\sigma_\alpha^2)$ with $\sigma=2$, $\mu_\alpha=0$, $\sigma_\alpha=3$.

Site A ($n=100, \bar y=5$):

$\hat\alpha_A = \dfrac{100/4 \cdot 5 + 1/9 \cdot 0}{100/4 + 1/9} = \dfrac{125}{25.11} \approx 4.98$

Pulled almost none.

Site B ($n=4, \bar y=5$):

$\hat\alpha_B = \dfrac{4/4 \cdot 5 + 1/9 \cdot 0}{4/4 + 1/9} = \dfrac{5}{1.111} \approx 4.50$

Pulled 0.5 units toward 0.

Punchline. Same observed mean, very different posterior estimates — because Site B's $\bar y$ is less precise. This is the entire mechanism of hierarchical models. Memorise this formula; it generalises to almost every random-intercept model you'll ever fit.

Exercise 6 — Beverton-Holt: fit, predict, derive

Run SpawnerRecruit_Nimble.R top to bottom.

(a) Posterior summaries (typical):

ParameterMedian95% CITruth
$a$~ 4.8[3.0, 7.2]5
$b$~ 4900[3500, 7100]5000
$\sigma$~ 0.21[0.15, 0.31]0.2
$N_\text{eq}$~ 3800[2400, 5800]4000
$S_\text{msy}$~ 1300[700, 2200]1316
$H_\text{msy}$~ 0.55[0.42, 0.68]0.553

(b) Conservative management value. 5th percentile of $H_\text{msy}$ from your quantile() call — typically around 0.40 with the default 20-year dataset. That is the "if I want to be 95% confident I'm not over-fishing" harvest rate.

(c) More data, tighter posteriors. With $N=100$ years, $H_\text{msy}$ CI half-width shrinks by roughly $\sqrt{5} \approx 2.2$. Bayesian inference obeys the same $\sqrt{N}$ rule as frequentist inference at large $N$; the prior matters only at small $N$.

Instructor note. Some students will get NaN log() values on iteration 1 if the random initial $a$ leads to a negative argument inside log(1 + a/b * S[t]). The prior bounds dunif(1, 10) and dunif(1000, 10000) usually prevent this; if not, supply explicit inits.

Exercise 7 — Dynamic occupancy

Run DynamicOccupancy_Nimble.R.

(a-c) Posterior medians for $\phi_t, \gamma_t, \psi_t$ plotted vs truth: scattered around the 1:1 line, but with large per-parameter uncertainty due to small sample size (30 sites, 3 reps, 7 transitions).

(d) Without z-init trick. Try:

bad.z <- matrix(rbinom(Nsites*Nyears, 1, 0.3), Nsites, Nyears)
inits.bad <- list(list(z=bad.z, phi=runif(Nyears-1,.1,.9),
 gamma=runif(Nyears-1,.1,.9), p=.5, psi=.4))
nimbleMCMC(DynOccCode, constants=constants, data=data, inits=inits.bad, ...)

NIMBLE will report:

Error: Dynamic Index out of bounds, or initial values produce a density of 0

or similar — because for any (i,t) where the random $z[i,t] = 0$ but the data has $y[i,j,t] = 1$, the Bernoulli likelihood is 0 and $\log 0 = -\infty$. Always initialise $z$ at 1.

(e) Why CIs are wide. Each transition $t \to t+1$ has only 30 sites of evidence. A standard fix is to give $\phi, \gamma$ themselves a hierarchical prior (e.g. logit$(\phi_t) \sim \mathcal{N}(\mu_\phi, \sigma_\phi^2)$), which lets years borrow strength from each other and shrinks year-specific estimates. You can also add covariates like temperature or land-use change to explain year-to-year variation.

Exercise 8 — N-mixture build-up

(a) Basic. Posterior medians: $\lambda \approx 5.9$ [4.5, 7.6], $p \approx 0.78$ [0.62, 0.91]. Truth $\lambda=6, p=0.8$.

(b) Covariate on abundance. $\beta_1$ posterior median $\approx 1.0$ with 95% CI well away from 0. The recovered slope matches the simulation truth ($\beta_1 = 1$).

(c) Covariate on both. $\beta_1$ posterior is very similar to (b) here (because data are clean), but in general, ignoring detection heterogeneity inflates the apparent abundance heterogeneity. The slope estimates for both abundance and detection are well-recovered.

(d) Overdispersion + Bayesian p-value. Bayesian p-value typically < 0.05 or > 0.95 — clear evidence the Poisson model can't capture the variance. The point is that the discrepancy test catches a model that would otherwise produce confident-sounding (and wrong) posteriors for $\lambda$.

Exercise 9 — Add a Poisson-lognormal random effect to fix overdispersion

Nmix5Code <- nimbleCode({
 for (i in 1:Nsites) {
 eps[i] ~ dnorm(0, tau.eps)
 log(lambda[i]) <- b0 + eps[i]
 N[i] ~ dpois(lambda[i])
 for (j in 1:Nreps) {
 y[i,j] ~ dbinom(prob=p, size=N[i])
 y.pred[i,j] ~ dbinom(prob=p, size=N[i])
 e[i,j] <- p*lambda[i]
 resid[i,j] <- pow(pow(y[i,j], 0.5) - pow(e[i,j],0.5), 2)
 resid.pred[i,j] <- pow(pow(y.pred[i,j],0.5) - pow(e[i,j],0.5), 2)
 }
 }
 fit.data <- sum(resid[1:Nsites, 1:Nreps])
 fit.pred <- sum(resid.pred[1:Nsites, 1:Nreps])

 b0 ~ dunif(-5, 5)
 p ~ dbeta(1, 1)
 sig.eps ~ dunif(0, 5)
 tau.eps <- 1/sig.eps^2
})

Expected. Bayesian p-value moves toward 0.5 (usually 0.3-0.7). $\sigma_\epsilon$ posterior median is around 0.5-0.7 with 95% CI well away from 0, capturing the overdispersion the Poisson alone missed.

Conceptual. The Poisson-lognormal is the workhorse fix for overdispersed N-mixture data. Some authors prefer a negative-binomial latent abundance (N[i] ~ dnegbin(...)), which is equivalent in the limit but parameterised differently.
