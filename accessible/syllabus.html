FW 536 · 3 credits · Oregon State University

Statistical Modeling for Ecology and Conservation

A five-day intensive workshop held immediately before fall term. Builds the probability, GLM, mixed-model, maximum-likelihood, and Bayesian-modeling foundation that ecologists and conservation scientists need to analyze their own data.

Catalog course description

Explores statistical modeling and practical analysis of data typical of ecology, conservation, and natural resources. Facilitates deep interpretation and application by covering probability, continuous and discrete probability distributions and their use for ecology, generalized linear models and their interpretation, mixed effects models for ecological data, model selection, and applications of maximum likelihood and Bayesian statistics for common hierarchical modeling problems in ecology. Includes hands-on exercises in the R programming language and the BUGS language implemented in Nimble and JAGS.

Daily schedule

Each day has a morning lecture + morning lab and an afternoon lecture + afternoon lab. The morning lab is the assignment that goes with the morning lecture, completed and submitted before the next session; the afternoon lab is similar. Each lab is worth 20 points.

Course-specific measurable student learning outcomes

By the end of this course, students will be able to:

  1. Determine the appropriate probability distributions for distinct data types.
  2. Interpret and implement generalized linear models (GLMs) to analyze ecological and conservation data.
  3. Construct and evaluate generalized linear mixed-effects models (GLMMs) to address hierarchical ecological data.
  4. Implement model selection appropriately to evaluate multiple hypotheses for ecological data.
  5. Construct and evaluate nonlinear and hierarchical models common in the ecological and wildlife sciences in both maximum likelihood and Bayesian frameworks using JAGS and Nimble.

Submission format

Key idea

All assignments are submitted as R Markdown HTML outputs uploaded to Canvas. You can embed photos of hand-written derivations or diagrams directly into the R Markdown — many probability problems are easier to do on paper, and a photo of clean work plus a typed interpretation is acceptable for full credit.

An R Markdown tutorial on how to install R Markdown, knit to HTML, and embed photos is the first technical activity of Day 1 — please complete the install steps before the workshop begins so we don't lose time on setup.

Per-problem-set workflow

  1. Open the problem set in your browser to read the questions.
  2. Open the matching R Markdown template (provided alongside each problem set in the day's folder).
  3. Add your code, prose, and (if needed) photos of written work.
  4. Knit to HTML.
  5. Upload the resulting .html file to Canvas before the due time.

For each lab session, you'll see three problem-set documents:

  • One practice problem set with visible worked answers — this is what the instructor walks through during lab time.
  • Two assessed problem sets (v2 and v3) with answers in separate instructor-only files. You pick whichever theme fits your interests — the two versions cover the same statistical concepts in different ecological contexts (e.g., forest vs. aquatic, terrestrial vs. marine).

Pre- and post-course exams

The course includes a pre-course exam given at the start of Day 1 morning and a post-course exam given on Day 5 afternoon. The two exams use parallel questions covering all course content — probability, distributions, GLMs, mixed models, model selection, MLE, and Bayesian inference.

The pre/post pairing has two purposes:

  • Diagnostic at the start of the course. The pre-course exam tells the instructor where the class is starting from so the workshop can be paced and supplemented appropriately.
  • Measure of learning at the end of the course. The post-course exam, paired with the pre-course exam, lets us quantify how much each student gained.

You will not be penalized for low performance on the pre-course exam — its scoring is intentionally generous because you are not expected to know all of the material going in. The post-course exam is graded normally.

Learning resources

Students must have a laptop with the following installed and working:

  • R and RStudio.
  • JAGS.
  • A C++ compiler for Nimble — see installing Nimble.

Once JAGS is installed, install and load the R packages rjags, R2jags, and jagsUI to confirm that JAGS and R are interfacing. Install and load the nimble package and confirm Nimble is interfacing with R.

For R Markdown, install the rmarkdown and knitr packages (RStudio will prompt you the first time you create an Rmd file). Walking through the R Markdown tutorial is the first activity of Day 1.

Highly recommended (not required) books:

  • Hobbs & Hooten (2015). Bayesian Models: A Statistical Primer for Ecologists. Princeton.
  • Kéry & Schaub (2012). Bayesian Population Analysis using WinBUGS. Academic Press.
  • Bolker (2008). Ecological Models and Data in R. Princeton.
  • McElreath (2020). Statistical Rethinking. CRC Press.
