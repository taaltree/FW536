Day 3 · Alternate Problem Set v2

Terrestrial wildlife scenarios — mixed models and model selection

Ten novel problems (five mixed models, five model selection) using cougars, owls, elk, deer, marten, grouse, bighorn, coyotes, pronghorn, and tortoises. Answers are revealed beneath each problem.

This problem set is designed to take roughly the same time as the main lab. Try each problem first, then check the answer. Where simulations are requested, exact numbers depend on the seed; "approximately" is in the spirit of the exercise.

Part 1 · Mixed models

Problem 1 · pooling decision

Cougar home-range size across mountain ranges

A wildlife biologist measures the 95% MCP home-range size (km$^2$) of 60 GPS-collared cougars, distributed across 8 mountain ranges (5–10 cougars per range). The biological question: is there a real difference in mean home-range size among mountain ranges?

  1. Write out three candidate models — complete pooling, no pooling (range as fixed effect), partial pooling (range as random effect) — in mathematical notation, including all coefficients and distributions.
  2. Which model would you fit if you wanted to make inferences about cougars in other mountain ranges (not in this dataset)? Why?
  3. For 4 of the ranges, only 5 cougars were collared; for the other 4 ranges, 10 cougars were collared. Will partial pooling shrink the 5-cougar ranges more or less than the 10-cougar ranges, and why?
Answer

(a)

  • Complete pooling: $y_i \sim \mathcal{N}(\beta_0, \sigma^2)$.
  • No pooling: $y_{ij} \sim \mathcal{N}(\alpha_j, \sigma^2)$, with one $\alpha_j$ per range $j = 1,\ldots,8$.
  • Partial pooling: $y_{ij} \sim \mathcal{N}(\beta_0 + b_{0,j}, \sigma^2)$, $b_{0,j} \sim \mathcal{N}(0, \sigma_R^2)$.

(b) Partial pooling. The 8 ranges are exchangeable samples from a population of mountain ranges; the random-effect variance $\sigma_R^2$ tells you how much among-range variability to expect when predicting for a new range.

(c) The 5-cougar ranges get pulled more toward the grand mean — fewer observations means less data-driven leverage, so the prior (the among-range distribution centered on $\beta_0$) dominates. Approximate shrinkage factor: $\sigma_R^2 / (\sigma_R^2 + \sigma^2/n_j)$, so smaller $n_j$ → more pull.

# Sketch of the lmer fit
library(lme4)
fit <- lmer(HR_km2 ~ 1 + (1 | Range), data=cougars)
VarCorr(fit)
ranef(fit)$Range # shrinkage visible: small-n ranges sit closer to 0
Problem 2 · random intercepts + covariate

Spotted owl pair productivity across forests

Over 6 years you monitor breeding success of 240 spotted owl pairs nested in 24 national forests. Each pair has its territory size (ha) measured. The response is fledglings produced. You fit:

m <- lmer(Fledglings ~ scale(Territory_ha) + (1 | Forest), data=owls)
summary(m)
# Random effects:
# Groups Name Variance Std.Dev.
# Forest (Intercept) 0.18 0.42
# Residual 0.92 0.96
# Fixed effects:
# Estimate Std.Error t
# (Intercept) 1.32 0.12 11.0
# scale(Territory_ha) 0.31 0.07 4.4
  1. Write out the equation the model is fitting.
  2. What is the ICC? Translate it into a sentence about what fraction of variation is "between forests" vs "among pairs within a forest."
  3. Compute the expected fledgling count for a pair with average territory size in an "average" forest, and the same in a forest one SD below the population mean intercept.
  4. Is fledgling count really Normal? What family would you switch to and why?
Answer

(a) $F_{ij} = \beta_0 + b_{0,j} + \beta_1 \tilde T_{ij} + \varepsilon_{ij}$, $b_{0,j}\sim\mathcal{N}(0,0.18)$, $\varepsilon\sim\mathcal{N}(0,0.92)$, where $\tilde T$ is z-scored territory size.

(b) ICC = $0.18/(0.18+0.92) = 0.164$. About 16% of remaining (after territory) variation in fledglings is between forests, 84% is among pairs within forests.

(c) Average territory, average forest: $1.32 + 0.31 \times 0 = 1.32$ fledglings. Forest one SD below average: $1.32 - 0.42 = 0.90$ fledglings.

(d) Fledgling counts are non-negative integers — should be Poisson or Negative Binomial. Switch to glmer(Fledglings ~ scale(Territory_ha) + (1|Forest), family=poisson) or glmmTMB(..., family=nbinom2). The current Normal fit is approximate; on counts < 5 the Normal assumption is rough and gives non-zero probability to negative fledgling counts.

Problem 3 · random slope

Elk body mass vs age, across herds

Body mass (kg) is measured on 480 elk across 12 herds. The biological question: how does body mass scale with age, and does the slope differ by herd?

m_int <- lmer(Mass ~ Age + (1 | Herd), data=elk, REML=TRUE)
m_slope <- lmer(Mass ~ Age + (1 + Age | Herd), data=elk, REML=TRUE)
anova(m_int, m_slope, refit=FALSE)
# Df AIC BIC logLik Chisq Chi Df Pr(>Chisq)
# 4 ... ... ... ...
# 6 ... ... ... 14.6 2 0.0007
  1. What does the random slope mean biologically? Write the implied bivariate Normal for the (intercept, slope) pair per herd.
  2. Interpret the LRT. Why refit = FALSE?
  3. What would a singular-fit warning here likely mean?
Answer

(a) Different herds have different age-mass slopes (e.g., due to forage quality, predation pressure, body-condition trajectories). The pair $(b_{0,j}, b_{1,j})$ is drawn from a bivariate Normal $\mathcal{N}(\mathbf{0}, \Sigma)$ with $\Sigma = \begin{pmatrix}\sigma_0^2 & \rho\sigma_0\sigma_1 \\ \rho\sigma_0\sigma_1 & \sigma_1^2\end{pmatrix}$.

(b) $\chi^2 = 14.6$ on 2 df, $p = 0.0007$ — strong support for the random slope. The 2 df = 1 (slope variance) + 1 (slope-intercept correlation). REML log-likelihoods are comparable because the fixed-effect specification is identical; we use REML for unbiased variance components and pass refit = FALSE to keep both fits on REML.

(c) A singular fit would indicate that the slope variance is estimated at zero, or the slope-intercept correlation is at $\pm 1$. Possible causes: too few age values per herd to identify both slope and intercept variability; small among-herd slope differences. Diagnose with VarCorr(m_slope).

Problem 4 · nested random effects

Mule deer aerial counts nested in management units

The Oregon Department of Fish & Wildlife flies aerial surveys: 3 flights × 6 transects per flight × 4 management units = 72 transect-flights, each producing a deer count. Counts vary among units, among transects within units, and among flights within transects.

  1. Write out the equation for a Poisson GLMM with a nested random structure (unit / transect / flight is not really needed since flights are within transects within units — diagram the nesting first).
  2. What is the difference between (1 | unit/transect) and (1 | unit) + (1 | transect) in lme4 syntax?
  3. If transect IDs are reused across units (transect "1" exists in every unit but they're different transects), which syntax should you use?
  4. For the Poisson GLMM, how do you check for overdispersion, and what do you do about it?
Answer

(a) $\log \lambda_{ijk} = \beta_0 + b_{0,k} + b_{0,jk} + b_{0,ijk}$, $b_{0,k}\sim\mathcal{N}(0,\sigma_U^2)$ (unit), $b_{0,jk}\sim\mathcal{N}(0,\sigma_T^2)$ (transect within unit), $b_{0,ijk}\sim\mathcal{N}(0,\sigma_F^2)$ (flight within transect within unit; observation-level if only one count per flight). With one count per flight there is no flight-level residual to estimate separately from Poisson sampling variation — drop it or use it explicitly as an observation-level random effect to absorb overdispersion.

(b) (1|unit/transect) expands to (1|unit) + (1|unit:transect) — i.e., it creates a unique random effect per unit-transect combination, which is what you want when transect IDs are reused across units. (1|unit) + (1|transect) treats transect IDs as crossed with unit — only correct if "transect 1" really is the same physical transect across units.

(c) Use (1|unit/transect).

(d) Pearson dispersion $\hat\phi = \sum r_i^2 / (n-p)$ — should be near 1. If $\hat\phi > 1.5$, the data are overdispersed. Solutions: switch to glmmTMB(..., family=nbinom2) (Negative Binomial), or add an observation-level random effect to soak up extra variability.

library(glmmTMB)
m <- glmmTMB(Deer ~ 1 + (1 | unit/transect), data=surveys, family=poisson)
sum(residuals(m, type="pearson")^2) / df.residual(m) # dispersion check
Problem 5 · variance partitioning + interpretation

Pacific marten detection probabilities across regions

You fit a logistic mixed model to camera-trap data:

m <- glmer(Detect ~ scale(SnowDepth) + (1 | Region/Camera), data=marten, family=binomial)
VarCorr(m)
# Groups Name Variance Std.Dev.
# Camera:Region (Intercept) 1.05 1.02
# Region (Intercept) 0.46 0.68
# Fixed effects:
# Estimate Std.Error z
# (Intercept) -1.4 0.30 -4.6
# scale(SnowDepth) 0.62 0.13 4.8
  1. On the logit scale, what fraction of total non-fixed variation lives between regions vs between cameras within regions vs residual? (For a binomial logit, the implicit residual variance is $\pi^2/3 \approx 3.29$.)
  2. For a camera with mean snow depth and average region/camera intercepts, what is the expected detection probability?
  3. Translate the SnowDepth slope into an odds ratio per 1 SD of snow depth.
  4. Why do we add the $\pi^2/3$ term in the ICC formula for a binomial GLMM?
Answer

(a) Total = $0.46 + 1.05 + 3.29 = 4.80$. Region: $0.46/4.80 = 9.6\%$. Camera-within-region: $1.05/4.80 = 21.9\%$. Residual (binomial sampling): $3.29/4.80 = 68.5\%$.

(b) $\text{plogis}(-1.4) = 1/(1+e^{1.4}) \approx 0.198$. About a 20% detection probability per visit at average conditions.

(c) Odds ratio = $e^{0.62} \approx 1.86$. One SD more snow → odds of detection multiplied by ~1.86. The 95% CI: $e^{0.62 \pm 1.96 \times 0.13} \approx (1.45, 2.40)$ — strong evidence for a positive effect of snow depth.

(d) Binomial GLMMs have no separate residual variance parameter — the Bernoulli/Binomial sampling variability is the residual. On the latent logit scale, the implied residual variance of a Bernoulli draw is $\pi^2/3$, and including it in the denominator lets you compute an ICC that is comparable to the linear-Gaussian ICC.

Part 2 · Model selection

Problem 6 · LRT vs AICc

Sage-grouse lek attendance: nested fixed-effect comparison

You have 8 years of lek-attendance counts on 30 leks. Candidate fixed effects: year (continuous), spring_precip, and their interaction. Lek is a random intercept.

m0 <- lmer(Attend ~ 1 + (1|Lek), REML=FALSE, data=g)
m1 <- lmer(Attend ~ year + (1|Lek), REML=FALSE, data=g)
m2 <- lmer(Attend ~ year + spring_precip + (1|Lek), REML=FALSE, data=g)
m3 <- lmer(Attend ~ year * spring_precip + (1|Lek), REML=FALSE, data=g)

anova(m0, m1, m2, m3)
# Df AIC BIC logLik deviance Chisq Chi Df Pr(>Chisq)
# m0 3 ... ... ...
# m1 4 ... ... ... 8.2 1 0.004
# m2 5 ... ... ... 5.8 1 0.016
# m3 6 ... ... ... 1.2 1 0.27
  1. Why are all four models fit with REML = FALSE?
  2. What's the "best" model by sequential LRT?
  3. By AICc, m2 and m3 differ by approximately how much (use the LRT statistic for m2-vs-m3 plus the parameter count)? Does AICc agree with the LRT here?
  4. What's the danger of doing this kind of sequential testing as opposed to specifying a single principal model up front?
Answer

(a) We are comparing models that differ in fixed-effect structure. REML log-likelihoods are not comparable across different fixed effects, so we fit with ML.

(b) Sequential LRT: m1 supported over m0 ($p=0.004$), m2 supported over m1 ($p=0.016$), m3 not supported over m2 ($p=0.27$). Pick m2.

(c) Going from m2 to m3 adds 1 parameter; AIC penalty = 2; LRT statistic = 1.2 (twice the log-likelihood improvement). So $\Delta\text{AIC} = 2 - 1.2 = 0.8$ in favor of m2. AICc agrees with the LRT — the simpler model is better.

(d) Sequential testing inflates the family-wise Type I error rate; the resulting "best" model's coefficients and p-values are biased toward significance (the same garden of forking paths critique as dredging). Better: specify the principal model based on biology before seeing the data.

Problem 7 · dredge critique

Bighorn sheep lamb survival: a dredge gone wrong

A graduate student fits a logistic regression of lamb survival on 12 candidate predictors (elevation, slope, aspect, vegetation indices, distance-to-water, etc.) on 90 lambs. They run dredge() and find a top model containing 5 predictors with $\Delta\text{AICc} = 0$. They report the coefficients with their p-values.

  1. What is the biggest problem with this analysis?
  2. Simulate it: $n=90$, $y \sim \text{Bernoulli}(0.5)$ unrelated to 12 random $U(0,1)$ predictors. How often does the AICc-best model have ≥ 1 "significant" coefficient (p < 0.05) on a predictor?
  3. What workflow should they have used instead?
Answer

(a) Post-selection inference: the reported p-values come from the model selected from many candidates, so the coefficients are biased away from zero and the standard errors don't reflect the selection process. The naive p-values vastly understate the actual Type I error rate.

(b) Approximate simulation:

library(MuMIn); options(na.action = "na.fail")
set.seed(2026); reps <- 200
res <- replicate(reps, {
 n <- 90; y <- rbinom(n, 1, 0.5)
 X <- as.data.frame(matrix(runif(n*12), n, 12)); names(X) <- paste0("x",1:12)
 d <- cbind(y, X)
 gm <- glm(y ~ ., data=d, family=binomial)
 top <- get.models(dredge(gm, rank="AICc"), 1)[[1]]
 any(coef(summary(top))[-1, "Pr(>|z|)"] < 0.05)
})
mean(res) # typically ~ 0.85 - 0.95

The selected top model has at least one "significant" coefficient ~90% of the time, despite the data being pure noise.

(c) Pre-specify a small set of biologically motivated candidates; reserve out-of-sample data for validation; or use a regularized regression (LASSO/elastic net) that explicitly accounts for selection. If you must run an all-subsets selection, treat it as exploratory and validate the final model on a held-out fold.

Problem 8 · BIC vs AIC

Coyote pup survival: when BIC misses the weak effect

You want to know whether 3 predictors affect coyote pup survival: litter_size (strong known effect), maternal_age (modest effect), and winter_severity (weak effect). $n = 60$ pups. Simulate:

set.seed(7); n <- 60
litter <- runif(n); age <- runif(n); winter <- runif(n)
y <- 0.8*litter - 0.4*age + 0.1*winter + rnorm(n, 0, 0.5)
d <- data.frame(y, litter, age, winter)
m <- lm(y ~ litter + age + winter, data=d)
dredge(m, rank="AIC"); dredge(m, rank="AICc"); dredge(m, rank="BIC")
  1. Run the simulation. Which predictors does each criterion retain in the top model?
  2. Now wrap it in a 500-rep loop and report the proportion of times each criterion retains winter.
  3. Which criterion would you use if your science question is "is winter severity worth studying further?", and which if it's "does the winter signal generalize out of sample?"
Answer

(a) A typical realization: AIC and AICc retain all three; BIC retains only litter and age (drops winter).

(b) Approximate 500-rep proportions of retaining winter in the top model: AIC ≈ 0.55, AICc ≈ 0.50, BIC ≈ 0.15. BIC misses the weak signal most often.

(c) For the "worth studying further" question, prefer AICc — it has higher power to detect weak effects, and a worth-studying-further question explicitly admits the possibility of a weak signal. For the "generalize out of sample" question, prefer cross-validation; BIC's parsimony bias can make it more robust against overfitting to noise.

Problem 9 · LOOCV for prediction

Pronghorn fawn:doe ratios — predicting next year's index

You have 15 years of pronghorn fawn:doe ratios, with annual covariates: june_precip, winter_snow, coyote_index. You want to predict next year's ratio.

library(boot)
forms <- list(
 null = ratio ~ 1,
 prec = ratio ~ june_precip,
 prec_snow = ratio ~ june_precip + winter_snow,
 full = ratio ~ june_precip + winter_snow + coyote_index)
cv <- sapply(forms, function(f) cv.glm(d, glm(f, data=d))$delta[1])
aic <- sapply(forms, function(f) AIC(glm(f, data=d)))
data.frame(cv, aic)
  1. With $n = 15$ years and 4 candidate models, what is the practical concern with LOOCV in this setting?
  2. Why does AICc (rather than AIC) become especially important here?
  3. If the LOOCV ranking and the AICc ranking disagree, which would you trust for the prediction task and why?
  4. Suggest one diagnostic for whether either criterion's "best" model is actually any good in absolute terms.
Answer

(a) $n=15$ is small. LOOCV variance can be large; one or two influential years drive the result. Consider $k$-fold CV with $k=5$ if you have enough data, or report LOOCV SE alongside the point estimate.

(b) AICc adds a $2k(k+1)/(n-k-1)$ correction that matters when $n/k$ is small. With $n=15$ and up to $k=4$ predictors, the correction is substantial (≈ 4 added to AIC for the full model). Without it, AIC will tend to favor overfit models.

(c) Trust LOOCV for a prediction question. AICc is a theoretical estimate of out-of-sample performance; CV is a direct measurement. If they disagree, the data are telling you the asymptotic AICc approximation isn't accurate at $n=15$.

(d) Compare the LOOCV MSE of the best model to the variance of the response itself ($\text{Var}(y)$). A model whose CV MSE is barely smaller than $\text{Var}(y)$ is doing almost no useful work — even if it's the "best" of the bunch.

Problem 10 · multimodel inference

Desert tortoise survival: when not to model-average

You fit 6 candidate logistic models of tortoise survival. Three have nearly identical AICc ($\Delta\text{AICc} < 1$); the rest are far behind. A reviewer asks for "model-averaged predictions" using Akaike weights. The three competitive models contain different combinations of correlated predictors (NDVI, annual_rainfall, summer_high_temp).

  1. How are Akaike weights computed from $\Delta\text{AICc}$? Write the formula and the weights for $\Delta = 0, 0.4, 0.9, 4.2, 8.7, 15.1$.
  2. Why is model-averaging coefficients potentially misleading when the candidate models contain different (correlated) predictors?
  3. Why is model-averaging predictions usually safer than averaging coefficients?
  4. What single number would you also report so a reader can judge how much of the inference is being driven by one of the three competitive models?
Answer

(a) $w_i = \exp(-\tfrac{1}{2}\Delta_i) / \sum_j \exp(-\tfrac{1}{2}\Delta_j)$.

delta <- c(0, 0.4, 0.9, 4.2, 8.7, 15.1)
w <- exp(-0.5*delta); w <- w / sum(w)
round(w, 3)
# 0.359 0.294 0.229 0.044 0.005 0.000 (intercept-only model has ~0 weight)

(b) A coefficient on (say) NDVI has different meaning in a model that also contains annual_rainfall vs a model that doesn't — because they are correlated. Averaging the two coefficient estimates conflates partial effects under different conditional structures.

(c) Predictions are on the response scale and are well-defined regardless of which predictors are in the model. Each model gives a prediction; you average those predictions with their weights, and the weighted prediction makes sense even if the underlying coefficients don't.

(d) Report the Akaike weight of the top model (or the top model's weight $\div$ the second's), so readers can see whether one model dominates. Bonus: also report predictions from the best model alone; if they barely differ from the weighted predictions, model averaging is not adding much.
