Day 5 · Problem Set v3

Anadromous fish & freshwater ecology — steelhead, sockeye, bull trout, salmon redds

Ten Bayesian problems in NIMBLE applied to fish: spawner-recruit, redd-based abundance, dynamic occupancy, hierarchical survival, derived viability quantities. Reveal the answers as you go.

Plain-language summary

Salmonid management is essentially a series of derived-quantity questions: how many spawners produce maximum sustained yield, how likely is the population to fall below 100 spawners in 20 years, what is the recovery time for a recolonising tributary. Bayesian models with latent biological states and explicit derived quantities map onto these decisions far better than point-estimate frequentist fits.

Problem 1 · 20 min · hierarchical Poisson

Steelhead redd counts across 15 tributaries

Total redds counted per tributary in a single survey year are modeled as:

$y_j \sim \text{Poisson}(L_j \cdot \rho_j), \quad \log \rho_j \sim \mathcal{N}(\mu, \sigma^2)$

where $L_j$ = surveyed stream length (km) and $\rho_j$ = redds per km in tributary $j$. Fit in NIMBLE.

Reveal solution and code
library(nimble); library(MCMCvis); library(coda)
set.seed(11)
nT <- 15
L <- runif(nT, 2, 20)
mu.t <- log(3); sd.t <- 0.6
log.rho <- rnorm(nT, mu.t, sd.t)
y <- rpois(nT, L * exp(log.rho))

reddCode <- nimbleCode({
 for (j in 1:nT) {
 log.rho[j] ~ dnorm(mu, tau)
 y[j] ~ dpois(L[j] * exp(log.rho[j]))
 }
 mu ~ dnorm(0, 1/4)
 sigma ~ dunif(0, 5); tau <- 1/sigma^2
})
inits <- rep(list(list(mu=1, sigma=0.5, log.rho=rep(1, nT))), 3)
samp <- nimbleMCMC(reddCode, constants=list(nT=nT, L=L), data=list(y=y),
 inits=inits,
 monitors=c("mu","sigma","log.rho"),
 nchains=3, niter=10000, nburnin=1000,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(samp, params=c("mu","sigma"), round=3)

Interpretation. Population-level mean log-density $\mu$ and between-tributary SD $\sigma$ are recovered. Tributary-specific $\rho_j$ are pulled toward $\exp(\mu)$ for low-survey-length tributaries (the offset structure means short surveys have little information). This is the classic offset-Poisson hierarchical model used in CTUIR, ODFW, and WDFW redd reporting.

Problem 2 · 30 min · Beverton-Holt + derived Smsy

Sockeye spawner-recruit, harvest at 95% confidence

You have 25 years of $(S_t, R_t)$ for a Bristol Bay sockeye stock. Fit Beverton-Holt and report $S_\text{msy}$, $H_\text{msy}$ posteriors. Then compute the 5th percentile harvest rate (the "precautionary" $H_\text{msy}$).

Reveal solution and code
set.seed(12)
Nyr <- 25; a.t <- 4; b.t <- 8000; sig.t <- 0.3
S <- rlnorm(Nyr, 8, 0.6)
R <- numeric(Nyr)
for (t in 1:Nyr) R[t] <- (a.t*S[t])/(1+a.t/b.t*S[t]) * exp(rnorm(1,0,sig.t))

bhCode <- nimbleCode({
 for (t in 1:Nyr) {
 log.mu[t] <- log(a) + log(S[t]) - log(1 + (a/b)*S[t])
 log.R[t] ~ dnorm(log.mu[t], tau)
 }
 a ~ dunif(1, 15); b ~ dunif(1000, 20000)
 sigma ~ dunif(0,2); tau <- 1/sigma^2
 Smsy <- b*sqrt(1/a) - b/a
 Hmsy <- 1 - sqrt(1/a)
})
inits <- rep(list(list(a=3, b=8000, sigma=0.3)), 3)
samp <- nimbleMCMC(bhCode, constants=list(Nyr=Nyr, S=S),
 data=list(log.R=log(R)), inits=inits,
 monitors=c("a","b","sigma","Smsy","Hmsy"),
 nchains=3, niter=20000, nburnin=2000,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(samp, round=3)
post <- as.matrix(samp)
quantile(post[, "Hmsy"], c(0.05, 0.5, 0.95))

Interpretation. The "5th-percentile $H_\text{msy}$" is the harvest rate you'd allow if you wanted only a 5% chance of overfishing. With $a_\text{true}=4$, the posterior median should be about 0.5; the 5th percentile around 0.4. Managers in Alaska use this exact computation for forecasting season targets.

Problem 3 · 25 min · prior sensitivity

Sockeye $a$ prior: does it change Smsy?

Re-fit Problem 2 with two different priors on $a$:

  1. Vague: $a \sim \text{Uniform}(1, 15)$ (the default)
  2. Informative: $a \sim \text{Lognormal}(\log 4, 0.3)$ (centered on 4, tight)

Compare posterior of $H_\text{msy}$. Comment on when the informative prior would and wouldn't be defensible.

Reveal solution and code
bhCode2 <- nimbleCode({
 for (t in 1:Nyr) {
 log.mu[t] <- log(a) + log(S[t]) - log(1 + (a/b)*S[t])
 log.R[t] ~ dnorm(log.mu[t], tau)
 }
 a ~ dlnorm(log(4), 1/0.09) # tighter: lognormal(log(4), 0.3)
 b ~ dunif(1000, 20000)
 sigma ~ dunif(0,2); tau <- 1/sigma^2
 Smsy <- b*sqrt(1/a) - b/a
 Hmsy <- 1 - sqrt(1/a)
})
samp2 <- nimbleMCMC(bhCode2, constants=list(Nyr=Nyr, S=S),
 data=list(log.R=log(R)), inits=inits,
 monitors=c("a","b","Hmsy"),
 nchains=3, niter=20000, nburnin=2000,
 samplesAsCodaMCMC=TRUE)
summary(as.matrix(samp)[, "Hmsy"])
summary(as.matrix(samp2)[, "Hmsy"])

Interpretation. The informative prior tightens the $H_\text{msy}$ posterior. When defensible: if meta-analysis of similar sockeye stocks gives you $a$ between 3-5 with high confidence. When NOT defensible: if the stock you're modeling is unique or the meta-analysis isn't trustworthy. Always pair an informative prior with a sensitivity analysis like this one.

Problem 4 · 30 min · hierarchical logistic

Bull trout occupancy across 8 basins

320 stream reaches in 8 basins, presence/absence data plus temperature (scaled). Model:

$\text{logit}\,p_i = \alpha_{j[i]} + \beta \cdot \text{temp}_i, \quad \alpha_j \sim \mathcal{N}(\mu_\alpha, \sigma_\alpha^2)$

Bull trout are temperature-limited; the slope $\beta$ should be negative.

Reveal solution and code
set.seed(14)
n <- 320; nB <- 8
basin <- sample(1:nB, n, replace=TRUE)
mu.a.t <- -0.5; sd.a.t <- 0.6; b.t <- -1.2
alpha.t <- rnorm(nB, mu.a.t, sd.a.t)
temp <- rnorm(n)
p.t <- plogis(alpha.t[basin] + b.t*temp)
y <- rbinom(n, 1, p.t)

bullCode <- nimbleCode({
 for (j in 1:nB) alpha[j] ~ dnorm(mu.a, tau.a)
 mu.a ~ dnorm(0, 1/2.71)
 sigma.a ~ dunif(0, 5); tau.a <- 1/sigma.a^2
 beta ~ dnorm(0, 1/2.71)
 for (i in 1:n) {
 logit(p[i]) <- alpha[basin[i]] + beta*temp[i]
 y[i] ~ dbern(p[i])
 }
})
inits <- rep(list(list(mu.a=0, sigma.a=1, beta=0, alpha=rep(0, nB))), 3)
samp <- nimbleMCMC(bullCode, constants=list(n=n, nB=nB, basin=basin, temp=temp),
 data=list(y=y), inits=inits,
 monitors=c("mu.a","sigma.a","beta","alpha"),
 nchains=3, niter=10000, nburnin=2000,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(samp, params=c("mu.a","sigma.a","beta"), round=3)

Interpretation. $\beta < 0$ with high confidence: warmer streams less likely to host bull trout. The basin-level random intercept controls for landscape-scale factors you didn't measure (geology, fragmentation history). Per-basin $\alpha_j$ posteriors are shrunk toward $\mu_\alpha$ for basins with fewer reaches.

Problem 5 · 30 min · N-mixture

Chinook smolt counts at screw traps

15 screw-trap sites, each operated nightly for 7 nights. Each night's count $y[i,j]$ is a binomial draw from latent nightly abundance $N_i$ (assumed constant within the week) with capture probability $p$.

Reveal solution and code
set.seed(15)
nS <- 15; reps <- 7
lam.t <- 50; p.t <- 0.15
N.t <- rpois(nS, lam.t)
y <- matrix(NA, nS, reps)
for (i in 1:nS) y[i, ] <- rbinom(reps, N.t[i], p.t)

smoltCode <- nimbleCode({
 for (i in 1:nS) {
 N[i] ~ dpois(lambda)
 for (j in 1:reps) {
 y[i,j] ~ dbinom(prob=p, size=N[i])
 }
 }
 lambda ~ dgamma(0.01, 0.01)
 p ~ dbeta(1, 1)
})
Ninit <- apply(y, 1, max)
inits <- rep(list(list(N=Ninit, lambda=30, p=0.2)), 3)
samp <- nimbleMCMC(smoltCode, constants=list(nS=nS, reps=reps),
 data=list(y=y), inits=inits,
 monitors=c("lambda","p"),
 nchains=3, niter=15000, nburnin=3000,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(samp, round=3)

Interpretation. Recovered $\lambda \approx 50$, $p \approx 0.15$. Compare to a simple "mean $\times$ correction-factor" approach — the Bayesian fit gives you a full uncertainty distribution on both abundance and capture probability without assuming the correction factor is known.

Problem 6 · 40 min · dynamic occupancy

Coho salmon recolonization after dam removal

45 reaches above a removed dam, surveyed for 5 years post-removal (3 replicate snorkels per year). You expect colonization $\gamma_t$ to rise as fish access new habitat. Persistence $\phi$ is high (coho return reliably once established).

Reveal solution and code
set.seed(16)
nS <- 45; nY <- 5; nR <- 3; p.t <- 0.55
phi.t <- c(0.85,0.85,0.85,0.85)
gam.t <- c(0.10,0.25,0.35,0.40)
z <- matrix(NA, nS, nY); z[,1] <- rbinom(nS, 1, 0.05)
for (i in 1:nS) for (t in 2:nY)
 z[i,t] <- rbinom(1, 1, z[i,t-1]*phi.t[t-1] + (1-z[i,t-1])*gam.t[t-1])
y <- array(NA, c(nS,nR,nY))
for (t in 1:nY) for (i in 1:nS) for (j in 1:nR)
 y[i,j,t] <- rbinom(1, 1, p.t)*z[i,t]

cohoCode <- nimbleCode({
 for (i in 1:nS) {
 z[i,1] ~ dbern(psi)
 for (t in 2:nY) {
 muZ[i,t] <- z[i,t-1]*phi[t-1] + (1-z[i,t-1])*gam[t-1]
 z[i,t] ~ dbern(muZ[i,t])
 }
 }
 for (t in 1:nY) for (i in 1:nS) for (j in 1:nR) {
 Py[i,j,t] <- z[i,t]*p
 y[i,j,t] ~ dbern(Py[i,j,t])
 }
 psi ~ dunif(0,1); p ~ dunif(0,1)
 for (t in 1:(nY-1)) { phi[t] ~ dunif(0,1); gam[t] ~ dunif(0,1) }
})
zst <- matrix(1L, nS, nY)
inits <- rep(list(list(z=zst, phi=runif(nY-1,.5,.9),
 gam=runif(nY-1,.1,.5), p=.5, psi=.1)), 3)
samp <- nimbleMCMC(cohoCode, constants=list(nS=nS, nY=nY, nR=nR),
 data=list(y=y), inits=inits,
 monitors=c("phi","gam","p","psi"),
 nchains=3, niter=20000, nburnin=5000,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(samp, round=3)

Interpretation. The fitted $\gamma_t$ posterior medians should rise across years — the recolonization signal. Even with only 45 reaches and 5 years, the upward trend in $\gamma$ is detectable. Pair this with covariates (distance from dam, stream gradient) for a publishable analysis.

Problem 7 · 40 min · hierarchical CJS

PIT-tag survival of steelhead smolts across years

10 release years; in each year, 500 smolts are tagged and survival to adult return is observed. Tag-recovery data summarised as $r_y / R_y$ (recoveries over releases). Fit a binomial model with annual random effects on survival:

$r_y \sim \text{Binomial}(R_y, \phi_y), \quad \text{logit}\,\phi_y \sim \mathcal{N}(\mu, \sigma^2)$
Reveal solution and code
set.seed(17)
nY <- 10
R <- rep(500, nY)
mu.t <- qlogis(0.04); sd.t <- 0.5 # ~4% SAR with year variation
logit.phi.t <- rnorm(nY, mu.t, sd.t)
phi.t <- plogis(logit.phi.t)
r <- rbinom(nY, R, phi.t)

sarCode <- nimbleCode({
 for (y in 1:nY) {
 logit.phi[y] ~ dnorm(mu, tau)
 phi[y] <- ilogit(logit.phi[y])
 r[y] ~ dbinom(prob=phi[y], size=R[y])
 }
 mu ~ dnorm(0, 1/2.71)
 sigma ~ dunif(0, 5); tau <- 1/sigma^2
 phi.mean <- ilogit(mu) # derived population-mean SAR
})
inits <- rep(list(list(mu=-3, sigma=0.5, logit.phi=rep(-3, nY))), 3)
samp <- nimbleMCMC(sarCode, constants=list(nY=nY, R=R),
 data=list(r=r), inits=inits,
 monitors=c("mu","sigma","phi","phi.mean"),
 nchains=3, niter=10000, nburnin=2000,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(samp, params=c("mu","sigma","phi.mean"), round=3)

Interpretation. Hierarchical SAR (smolt-to-adult return) estimation is the standard tool in CBR/BPA reports. $\phi_\text{mean} \approx 0.04$ with annual variation $\sigma \approx 0.5$ on logit scale. Year-specific $\phi_y$ are shrunk slightly toward $\phi_\text{mean}$, which dampens noise in years with few returns.

Problem 8 · 25 min · derived quantity

Egg-to-fry survival from staged binomial

You released 5,000 eggs in each of 20 redds; in fall, you sampled emergence traps and observed $f_j$ fry per redd. Survival per egg is the latent quantity; model fry as binomial with hierarchical survival.

$f_j \sim \text{Binomial}(5000, s_j), \quad \text{logit}\,s_j \sim \mathcal{N}(\mu, \sigma^2)$

Derive the posterior of mean egg-to-fry survival $\bar s = \text{ilogit}(\mu)$.

Reveal solution and code
set.seed(18)
nR <- 20; eggs <- rep(5000, nR)
mu.t <- qlogis(0.10); sd.t <- 0.6
logit.s.t <- rnorm(nR, mu.t, sd.t)
s.t <- plogis(logit.s.t)
fry <- rbinom(nR, eggs, s.t)

eggCode <- nimbleCode({
 for (j in 1:nR) {
 logit.s[j] ~ dnorm(mu, tau)
 s[j] <- ilogit(logit.s[j])
 fry[j] ~ dbinom(prob=s[j], size=eggs[j])
 }
 mu ~ dnorm(0, 1/2.71)
 sigma ~ dunif(0, 5); tau <- 1/sigma^2
 s.mean <- ilogit(mu)
})
inits <- rep(list(list(mu=-2, sigma=0.5, logit.s=rep(-2, nR))), 3)
samp <- nimbleMCMC(eggCode, constants=list(nR=nR, eggs=eggs),
 data=list(fry=fry), inits=inits,
 monitors=c("mu","sigma","s.mean"),
 nchains=3, niter=8000, nburnin=1000,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(samp, round=3)

Interpretation. Recovered $\bar s \approx 0.10$ with much tighter CI than any single-redd estimate would give. The hierarchical structure absorbs the redd-to-redd variability and reports the population-level survival with the right uncertainty.

Problem 9 · 35 min · Bayesian p-value

Does the basic redd Poisson fit?

Take Problem 1's redd model and add posterior-predictive Bayesian p-value checks. Compare $T(y) = \text{var}(y)/\text{mean}(y)$ between data and posterior replicates — if the data are overdispersed, this ratio will exceed 1 (Poisson predicts 1).

Reveal solution and code
# Simulate clearly overdispersed redd data
set.seed(19)
nT <- 15; L <- runif(nT, 2, 20)
library(MASS)
y <- rnegbin(nT, mu=L*5, theta=1.5)

reddGOFCode <- nimbleCode({
 for (j in 1:nT) {
 log.rho[j] ~ dnorm(mu, tau)
 y[j] ~ dpois(L[j] * exp(log.rho[j]))
 y.pred[j] ~ dpois(L[j] * exp(log.rho[j]))
 }
 mu ~ dnorm(0, 1/4)
 sigma ~ dunif(0, 5); tau <- 1/sigma^2

 mean.y <- mean(y[1:nT])
 var.y <- sd(y[1:nT])^2
 mean.y.pr <- mean(y.pred[1:nT])
 var.y.pr <- sd(y.pred[1:nT])^2

 T.data <- var.y / mean.y
 T.pred <- var.y.pr / mean.y.pr
})
inits <- rep(list(list(mu=1, sigma=0.5, log.rho=rep(1, nT))), 3)
samp <- nimbleMCMC(reddGOFCode, constants=list(nT=nT, L=L),
 data=list(y=y), inits=inits,
 monitors=c("mu","sigma","T.data","T.pred"),
 nchains=3, niter=10000, nburnin=2000,
 samplesAsCodaMCMC=TRUE)
post <- as.matrix(samp)
bayes.p <- mean(post[, "T.pred"] > post[, "T.data"])
cat("Bayesian p-value:", round(bayes.p, 3))

Interpretation. If the data come from a NB (as simulated here), the Bayesian p-value will be near 0 (most posterior replicates are less variable than the observed data). Reject Poisson; refit with $\log \rho_j \sim \mathcal{N}(\mu, \sigma^2)$ at the OBSERVATION level (Poisson-lognormal) or with a negative-binomial likelihood.

Problem 10 · 45 min · viability derived quantity

Probability of quasi-extinction in 20 years

Combining the spawner-recruit fit from Problem 2 and the hierarchical SAR fit from Problem 7, propagate posterior uncertainty through a 20-year forward population projection. Report $P(N_{t=20} < 100)$ assuming current spawners $S_0 = 500$.

Reveal solution and code
# Use posterior draws from Problem 2 fit (samp_sr) and Problem 7 (samp_sar).
# For this exercise, generate a quick BH fit again and a quick SAR fit;
# then simulate forward.
set.seed(20)
sr.post <- as.matrix(samp) # reuse last fit, or fit a quick BH
# Suppose we have draws of a, b, sigma from Problem 2 in post.bh
# and draws of phi.mean from Problem 7 in post.sar.
# For demonstration assume we sampled in lab as:
# post.bh <- as.matrix(samp.from.problem2)[, c("a","b","sigma")]
# post.sar <- as.matrix(samp.from.problem7)[, "phi.mean"]
# Substitute synthetic posteriors here so the code is runnable standalone:
n.draw <- 2000
post.bh <- cbind(a=rnorm(n.draw, 4, 0.6),
 b=rnorm(n.draw, 8000, 1500),
 sigma=runif(n.draw, 0.2, 0.4))
post.sar <- rbeta(n.draw, 15, 300) # ~0.05 SAR

S0 <- 500; Tyr <- 20; H <- 0.3 # harvest rate scenario
N.tr <- numeric(n.draw)
for (k in 1:n.draw) {
 ak <- post.bh[k,"a"]; bk <- post.bh[k,"b"]; sk <- post.bh[k,"sigma"]
 phk <- post.sar[k]
 S <- S0
 for (t in 1:Tyr) {
 R <- (ak*S)/(1+ak/bk*S) * exp(rnorm(1, 0, sk))
 adults <- R * phk # cohort survives to return
 S <- adults * (1 - H) # after harvest
 }
 N.tr[k] <- S
}
mean(N.tr < 100) # P(quasi-extinction)
hist(N.tr, breaks=40, main="Spawners after 20 yrs at H=0.3",
 xlab="Spawners year 20")
abline(v=100, lwd=2, lty=2, col="red")

Interpretation. This is a posterior-driven Population Viability Analysis (PVA). Every quantity entering the projection (productivity $a$, capacity $b$, process noise $\sigma$, marine survival $\phi$) carries its full posterior uncertainty into the 20-year forecast. The reported $P(\text{quasi-extinction})$ is the kind of probability that the ESA listings actually require — and Bayes makes it natural.

Extensions: add scenarios with different harvest rates $H$ and find the largest $H$ for which $P(\text{quasi-extinction}) < 0.05$ — that is your "risk-based" harvest target. Most modern salmon recovery plans have this structure underneath.
