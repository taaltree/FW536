Day 5 · Problem Set v2

Forest carnivore ecology — bears, martens, fishers, wolverines, lynx

Ten Bayesian problems in NIMBLE: hierarchical Poisson, hierarchical logistic, dynamic occupancy, N-mixture, derived quantities, prior sensitivity. Reveal the answer to check yourself.

Plain-language summary

Every problem is self-contained: the data are simulated in the same code block as the model. Run the code blocks in order, then open the <details> to see the full worked solution and biological interpretation.

Forest carnivores are perfect Bayesian problems: rare, hard to detect, hierarchically grouped (by basin, by year, by camera array), and managed for population recovery where derived quantities (occupancy, density, survival) matter more than raw counts.

Problem 1 · 20 min · hierarchical Poisson

Marten detection counts on baited camera arrays

Twelve camera arrays across the Oregon coast range each ran for 30 nights. Total marten detections at array $j$ are modelled as:

$y_j \sim \text{Poisson}(E_j \cdot \lambda_j), \quad \log \lambda_j \sim \mathcal{N}(\mu, \sigma_\lambda^2)$

where $E_j = 30$ camera-nights is offset effort. Fit in NIMBLE.

Reveal solution and code
library(nimble); library(MCMCvis); library(coda)
set.seed(1)
# Simulate
n.arr <- 12
mu.true <- -2.0; sd.true <- 0.7
log.lam <- rnorm(n.arr, mu.true, sd.true)
E <- rep(30, n.arr)
y <- rpois(n.arr, E * exp(log.lam))

martenCode <- nimbleCode({
 for (j in 1:n.arr) {
 log(lambda[j]) <- log.lam[j]
 log.lam[j] ~ dnorm(mu, tau)
 y[j] ~ dpois(E[j] * lambda[j])
 }
 mu ~ dnorm(0, 1/4)
 sigma ~ dunif(0, 5)
 tau <- 1/sigma^2
})
consts <- list(n.arr=n.arr, E=E)
data <- list(y=y)
inits <- list(list(mu=-2, sigma=0.5, log.lam=rep(-2, n.arr)),
 list(mu=-1, sigma=1.0, log.lam=rep(-2, n.arr)),
 list(mu=-3, sigma=0.2, log.lam=rep(-2, n.arr)))
samp <- nimbleMCMC(martenCode, constants=consts, data=data, inits=inits,
 monitors=c("mu","sigma","lambda"),
 nchains=3, niter=10000, nburnin=1000,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(samp, params=c("mu","sigma"), round=3)

Interpretation. $\mu$ posterior median ≈ -2 (matches truth), $\sigma$ ≈ 0.7. The 12 array-specific $\lambda_j$ shrink toward $\exp(\mu)$ for low-detection arrays where evidence is weak. This is the standard "small-area estimation" use case for hierarchical models in carnivore monitoring.

Problem 2 · 25 min · logistic with random intercept

Fisher occupancy on the Olympic peninsula

200 stations, grouped in 25 watersheds. Detection of fisher at each station ($y_i \in \{0,1\}$) depends on canopy cover ($x_i$, scaled), with a watershed-level random intercept.

$\text{logit}\,p_i = \alpha_{j[i]} + \beta x_i, \quad \alpha_j \sim \mathcal{N}(\mu_\alpha, \sigma_\alpha^2)$
Reveal solution and code
set.seed(2)
n <- 200; n.w <- 25
ws <- sample(1:n.w, n, replace=TRUE)
mu.a.t <- -1; sd.a.t <- 0.8; b.t <- 1.0
alpha.t <- rnorm(n.w, mu.a.t, sd.a.t)
x <- rnorm(n, 0, 1)
p.t <- plogis(alpha.t[ws] + b.t*x)
y <- rbinom(n, 1, p.t)

fisherCode <- nimbleCode({
 for (j in 1:n.w) { alpha[j] ~ dnorm(mu.a, tau.a) }
 mu.a ~ dnorm(0, 1/2.71)
 sigma.a ~ dunif(0, 5); tau.a <- 1/sigma.a^2
 beta ~ dnorm(0, 1/2.71)
 for (i in 1:n) {
 logit(p[i]) <- alpha[ws[i]] + beta*x[i]
 y[i] ~ dbern(p[i])
 }
})
consts <- list(n=n, n.w=n.w, ws=ws, x=x)
data <- list(y=y)
inits <- list(list(mu.a=0, sigma.a=1, beta=0, alpha=rep(0, n.w)),
 list(mu.a=-1, sigma.a=0.5, beta=1, alpha=rep(-1, n.w)),
 list(mu.a=-2, sigma.a=2, beta=-1, alpha=rep(1, n.w)))
samp <- nimbleMCMC(fisherCode, constants=consts, data=data, inits=inits,
 monitors=c("mu.a","sigma.a","beta","alpha"),
 nchains=3, niter=10000, nburnin=2000,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(samp, params=c("mu.a","sigma.a","beta"), round=3)

Interpretation. $\beta$ posterior ≈ 1.0 (canopy effect recovered). $\sigma_\alpha$ ≈ 0.8 captures watershed heterogeneity. Watershed-level random intercept absorbs unmeasured spatial covariates — you can compare $\sigma_\alpha$ across study regions to ask whether some landscapes are "lumpier" than others in fisher distribution.

Problem 3 · 30 min · N-mixture

Wolverine repeat-count abundance at 40 plots

40 winter trail-camera plots in the North Cascades, each visited 4 times. Snow-track count $y[i,j]$ at plot $i$ visit $j$. True wolverine "use intensity" $N_i \sim \text{Poisson}(\lambda)$, with a per-visit detection $p$.

Reveal solution and code
set.seed(3)
n.plot <- 40; reps <- 4
lam.t <- 2.5; p.t <- 0.4
N.t <- rpois(n.plot, lam.t)
y <- matrix(NA, n.plot, reps)
for (i in 1:n.plot) y[i, ] <- rbinom(reps, N.t[i], p.t)

wolvCode <- nimbleCode({
 for (i in 1:n.plot) {
 N[i] ~ dpois(lambda)
 for (j in 1:reps) {
 y[i,j] ~ dbinom(prob=p, size=N[i])
 }
 }
 lambda ~ dgamma(0.01, 0.01)
 p ~ dbeta(1, 1)
})
Ninit <- apply(y, 1, max)
consts <- list(n.plot=n.plot, reps=reps)
inits <- list(list(N=Ninit, lambda=2, p=0.3),
 list(N=Ninit, lambda=5, p=0.5),
 list(N=Ninit, lambda=1, p=0.6))
samp <- nimbleMCMC(wolvCode, constants=consts, data=list(y=y),
 inits=inits, monitors=c("lambda","p"),
 nchains=3, niter=15000, nburnin=3000,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(samp, round=3)

Interpretation. Posterior medians: $\lambda \approx 2.5$, $p \approx 0.4$. The "right answer" is jointly recovered. Without repeat visits, $\lambda$ and $p$ would be unidentified (only $\lambda \cdot p$ is identified from a single visit) — this is the central reason replicated counts exist for rare carnivores.

Problem 4 · 40 min · dynamic occupancy

Canada lynx recolonization in Maine

50 grid cells surveyed annually for 6 years (3 replicate visits per cell-year). After a state-line predator-control change in year 2, you expect colonization to rise. Fit a dynamic occupancy model with year-specific $\phi_t$ and $\gamma_t$.

Reveal solution and code
set.seed(4)
nS <- 50; nY <- 6; nR <- 3; p.t <- 0.5
phi.t <- c(0.7, 0.7, 0.8, 0.85, 0.85)
gamma.t <- c(0.05, 0.15, 0.25, 0.30, 0.30) # rises post-policy

z <- matrix(NA, nS, nY); z[, 1] <- rbinom(nS, 1, 0.2)
for (i in 1:nS) for (t in 2:nY) {
 z[i,t] <- rbinom(1, 1,
 z[i,t-1]*phi.t[t-1] + (1-z[i,t-1])*gamma.t[t-1])
}
y <- array(NA, c(nS, nR, nY))
for (t in 1:nY) for (i in 1:nS) for (j in 1:nR)
 y[i,j,t] <- rbinom(1, 1, p.t)*z[i,t]

lynxCode <- nimbleCode({
 for (i in 1:nS) {
 z[i,1] ~ dbern(psi)
 for (t in 2:nY) {
 muZ[i,t] <- z[i,t-1]*phi[t-1] + (1-z[i,t-1])*gam[t-1]
 z[i,t] ~ dbern(muZ[i,t])
 }
 }
 for (t in 1:nY) for (i in 1:nS) for (j in 1:nR) {
 Py[i,j,t] <- z[i,t]*p
 y[i,j,t] ~ dbern(Py[i,j,t])
 }
 psi ~ dunif(0,1); p ~ dunif(0,1)
 for (t in 1:(nY-1)) {
 phi[t] ~ dunif(0,1); gam[t] ~ dunif(0,1)
 }
})
zst <- matrix(1L, nS, nY)
consts <- list(nS=nS, nY=nY, nR=nR)
inits <- list(list(z=zst, phi=runif(nY-1,.2,.8),
 gam=runif(nY-1,.1,.5), p=.5, psi=.3))
samp <- nimbleMCMC(lynxCode, constants=consts, data=list(y=y),
 inits=rep(inits, 3),
 monitors=c("phi","gam","p","psi"),
 nchains=3, niter=20000, nburnin=5000,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(samp, round=3)

Interpretation. The posterior median of $\gamma_t$ should track the rising true values; the change-point in policy shows up as a rising colonization rate. With only 50 cells and 5 transitions, individual $\gamma_t$ posteriors are wide; the qualitative trend is what's interpretable.

Problem 5 · 30 min · N-mixture with covariates

Black bear hair-snare counts across elevation

60 hair snare stations, 5 visits each. Abundance varies with elevation (a U-shape: peak at middle elevations); detection varies with visit weather.

$\log \lambda_i = \beta_0 + \beta_1 \text{elev}_i + \beta_2 \text{elev}_i^2, \quad \text{logit}\,p_{ij} = \alpha_0 + \alpha_1 \text{rain}_{ij}$
Reveal solution and code
set.seed(5)
nS <- 60; reps <- 5
elev <- scale(runif(nS, 200, 2000))[, 1]
rain <- matrix(rnorm(nS*reps), nS, reps)
b0<-2; b1<-0.5; b2<--0.8; a0<-0.5; a1<--0.7
lam.t <- exp(b0 + b1*elev + b2*elev^2)
N.t <- rpois(nS, lam.t)
p.t <- plogis(a0 + a1*rain)
y <- matrix(NA, nS, reps)
for (i in 1:nS) for (j in 1:reps)
 y[i,j] <- rbinom(1, N.t[i], p.t[i,j])

bearCode <- nimbleCode({
 for (i in 1:nS) {
 log(lambda[i]) <- b0 + b1*elev[i] + b2*elev[i]*elev[i]
 N[i] ~ dpois(lambda[i])
 for (j in 1:reps) {
 logit(p[i,j]) <- a0 + a1*rain[i,j]
 y[i,j] ~ dbinom(prob=p[i,j], size=N[i])
 }
 }
 b0 ~ dnorm(0, 1/4); b1 ~ dnorm(0, 1/4); b2 ~ dnorm(0, 1/4)
 a0 ~ dnorm(0, 1/2.71); a1 ~ dnorm(0, 1/2.71)
})
Ninit <- apply(y, 1, max)
consts <- list(nS=nS, reps=reps, elev=elev, rain=rain)
inits <- rep(list(list(N=Ninit, b0=1, b1=0, b2=0, a0=0, a1=0)), 3)
samp <- nimbleMCMC(bearCode, constants=consts, data=list(y=y), inits=inits,
 monitors=c("b0","b1","b2","a0","a1"),
 nchains=3, niter=20000, nburnin=5000, thin=5,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(samp, round=3)

Interpretation. Coefficient $b_2 < 0$ recovers the U-shape (concave-down in elevation), $a_1 < 0$ confirms rain reduces detection. Without modeling both, you'd misinterpret detection variation as abundance variation — a classic mistake in carnivore monitoring data.

Problem 6 · 30 min · derived quantities

Total bear density across the survey region

Building on Problem 5, you want the posterior of total abundance $N_\text{tot} = \sum_i N_i$, and density $D = N_\text{tot} / A$ where $A = 600$ km².

Reveal solution and code
# Add derived quantities to bearCode; the rest stays identical
bearCode2 <- nimbleCode({
 for (i in 1:nS) {
 log(lambda[i]) <- b0 + b1*elev[i] + b2*elev[i]*elev[i]
 N[i] ~ dpois(lambda[i])
 for (j in 1:reps) {
 logit(p[i,j]) <- a0 + a1*rain[i,j]
 y[i,j] ~ dbinom(prob=p[i,j], size=N[i])
 }
 }
 b0 ~ dnorm(0, 1/4); b1 ~ dnorm(0, 1/4); b2 ~ dnorm(0, 1/4)
 a0 ~ dnorm(0, 1/2.71); a1 ~ dnorm(0, 1/2.71)

 # Derived quantities
 N.tot <- sum(N[1:nS])
 density <- N.tot / A
})
consts <- c(consts, list(A=600))
samp2 <- nimbleMCMC(bearCode2, constants=consts, data=list(y=y), inits=inits,
 monitors=c("b0","b1","b2","a0","a1","N.tot","density"),
 nchains=3, niter=20000, nburnin=5000, thin=5,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(samp2, params=c("N.tot","density"), round=3)

Interpretation. You get the posterior for total bears across the 60 stations and for density per km² with full uncertainty propagation. Note: the latent $N_i$ at sites you didn't visit are NOT in this sum — for a population estimate over the broader landscape, simulate $N_i$ at unsurveyed sites from the fitted $\lambda$ model (a posterior-predictive extension that's a one-line addition to the model).

Problem 7 · 25 min · prior sensitivity

Marten detection: does the prior matter?

Re-fit Problem 1 with three priors on $\mu$:

  1. Vague: $\mu \sim \mathcal{N}(0, \sigma^2 = 10000)$
  2. Weakly informative: $\mu \sim \mathcal{N}(0, \sigma^2 = 4)$
  3. Informative (centered on prior belief from previous studies): $\mu \sim \mathcal{N}(-2, \sigma^2 = 0.25)$

Compare the posterior of $\mu$ for each.

Reveal solution and code
fit_one <- function(prior.tau) {
 code <- nimbleCode({
 for (j in 1:n.arr) {
 log.lam[j] ~ dnorm(mu, tau)
 y[j] ~ dpois(E[j] * exp(log.lam[j]))
 }
 mu ~ dnorm(mu0, prior.tau)
 sigma ~ dunif(0, 5); tau <- 1/sigma^2
 })
 nimbleMCMC(code, constants=list(n.arr=n.arr, E=E,
 mu0=if(prior.tau>0.5) -2 else 0,
 prior.tau=prior.tau),
 data=list(y=y),
 monitors="mu", nchains=3, niter=5000, nburnin=1000,
 samplesAsCodaMCMC=TRUE)
}
fitA <- fit_one(1/10000)
fitB <- fit_one(1/4)
fitC <- fit_one(1/0.25)
sapply(list(A=fitA, B=fitB, C=fitC),
 function(s) summary(s)$statistics["Mean"])

Interpretation. Posteriors A and B should be very similar (the data are informative for $\mu$ with 12 arrays). Posterior C pulls toward -2 only if the data are sparse or noisy. Lesson: prior sensitivity matters most when data are weak (few groups, low counts, short time series). Always run this test on real analyses.

Problem 8 · 40 min · multi-species hierarchical

Five carnivores, shared detection structure

You ran 80 cameras for 30 nights and recorded presence/absence of each of 5 species (marten, fisher, bobcat, coyote, black bear). Fit a multi-species occupancy model with species-specific occupancy $\psi_s$ and detection $p_s$, both modeled hierarchically across species.

$\text{logit}\,\psi_s \sim \mathcal{N}(\mu_\psi, \sigma_\psi^2), \quad \text{logit}\,p_s \sim \mathcal{N}(\mu_p, \sigma_p^2)$
Reveal solution and code
set.seed(8)
nS <- 80; reps <- 10; nSp <- 5
psi.t <- c(0.20,0.15,0.40,0.50,0.25)
p.t <- c(0.10,0.08,0.20,0.30,0.12)
z <- sapply(psi.t, function(p) rbinom(nS, 1, p))
y <- array(NA, c(nS, reps, nSp))
for (s in 1:nSp) for (i in 1:nS) for (j in 1:reps)
 y[i,j,s] <- rbinom(1, 1, p.t[s])*z[i,s]

mscCode <- nimbleCode({
 for (s in 1:nSp) {
 lpsi[s] ~ dnorm(mu.psi, tau.psi)
 lp[s] ~ dnorm(mu.p, tau.p)
 psi[s] <- ilogit(lpsi[s])
 p[s] <- ilogit(lp[s])
 }
 for (s in 1:nSp) {
 for (i in 1:nS) {
 z[i,s] ~ dbern(psi[s])
 for (j in 1:reps) {
 py[i,j,s] <- z[i,s]*p[s]
 y[i,j,s] ~ dbern(py[i,j,s])
 }
 }
 }
 mu.psi ~ dnorm(0, 1/2.71); sigma.psi ~ dunif(0,5); tau.psi <- 1/sigma.psi^2
 mu.p ~ dnorm(0, 1/2.71); sigma.p ~ dunif(0,5); tau.p <- 1/sigma.p^2
})
zst <- matrix(1L, nS, nSp)
inits <- rep(list(list(z=zst, lpsi=rep(0, nSp), lp=rep(0, nSp),
 mu.psi=0, sigma.psi=1,
 mu.p=0, sigma.p=1)), 3)
samp <- nimbleMCMC(mscCode, constants=list(nS=nS, reps=reps, nSp=nSp),
 data=list(y=y), inits=inits,
 monitors=c("psi","p","mu.psi","mu.p","sigma.psi","sigma.p"),
 nchains=3, niter=15000, nburnin=5000,
 samplesAsCodaMCMC=TRUE)
MCMCsummary(samp, params=c("psi","p"), round=3)

Interpretation. Rare species (low $\psi_s$ or low $p_s$) borrow strength from the species-pool via the hyperparameters. This is the standard Dorazio-Royle multi-species occupancy framework. Adding a "data augmentation" step lets you also infer how many additional species in the community were never detected at all.

Problem 9 · 25 min · derived quantity

Fisher recolonization: time to 80% occupancy

Given posterior draws of $\phi, \gamma$ from a dynamic occupancy fit (use the values from Problem 4 for lynx as a proxy), derive the posterior distribution of the equilibrium occupancy $\psi_\infty = \gamma / (\gamma + 1 - \phi)$ and the years to reach 80% of equilibrium starting from $\psi_0 = 0.1$.

Reveal solution and code
# Suppose phi.post and gam.post are vectors of MCMC draws from Problem 4,
# with say phi.post = mean of phi[1..5] and same for gam.post.
phi.post <- rowMeans(as.matrix(samp)[, grep("^phi", colnames(as.matrix(samp)))])
gam.post <- rowMeans(as.matrix(samp)[, grep("^gam", colnames(as.matrix(samp)))])
psi.inf <- gam.post / (gam.post + 1 - phi.post)

# Time to 80% of equilibrium
# Closed form: psi_t = psi.inf + (psi0 - psi.inf) * (phi - 1 + gam)^t
# Solve psi_t = 0.8*psi.inf for t
psi0 <- 0.1
r <- phi.post - 1 + gam.post # the geometric rate
T80 <- log((0.8*psi.inf - psi.inf) / (psi0 - psi.inf)) / log(r)
summary(psi.inf); summary(T80)
hist(T80, breaks=40, main="Years to 80% of equilibrium occupancy")

Interpretation. The posterior of $\psi_\infty$ is one derived quantity; the posterior of "years to 80% recovery" is another. Both quantities have full uncertainty propagation. Policy decisions ("how long until we expect lynx to be common in this landscape") become probability statements, not point estimates.

Problem 10 · 45 min · multi-species indicator

Are habitat-specialist carnivores faring worse than generalists?

Using the multi-species occupancy fit from Problem 8, compute the posterior probability that the average occupancy of habitat specialists (species 1 = marten, species 2 = fisher) is less than the average occupancy of generalists (species 3 = bobcat, 4 = coyote, 5 = bear).

Reveal solution and code
post <- as.matrix(samp)
psi.specialists <- (post[,"psi[1]"] + post[,"psi[2]"]) / 2
psi.generalists <- (post[,"psi[3]"] + post[,"psi[4]"] + post[,"psi[5]"]) / 3
prob.spec.lower <- mean(psi.specialists < psi.generalists)
cat("P(specialists < generalists) =", prob.spec.lower)
hist(psi.generalists - psi.specialists, breaks=40,
 xlab="Generalist - specialist mean occupancy",
 main="")
abline(v=0, lwd=2, lty=2)

Interpretation. This kind of "probability that one composite group exceeds another" statement is direct in Bayes — just compute the relevant function at every MCMC iteration and tally. The frequentist equivalent (a likelihood-ratio test on a constrained vs unconstrained model) is clunkier and less directly interpretable. The combination of hierarchical multi-species + derived composite-index posterior is the Bayesian template for biodiversity assessment.
